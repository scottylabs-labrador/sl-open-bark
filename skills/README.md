# skills/

Portable **skill** folders: cross-runtime knowledge (instructions plus resources) a model loads
when relevant. A skill is the runtime-agnostic form of a workflow — where a recipe is Goose-native,
a skill is a portable folder a contributor can carry to any client (design Section 7, Appendix A).

**Owner:** per-committee (via CODEOWNERS).

Keep skills small and self-contained. Like recipes, a skill grants no new access; it composes
capabilities that already exist behind the gateway. Most committee workflows should be recipes;
reach for a skill when the knowledge needs to travel across runtimes.
